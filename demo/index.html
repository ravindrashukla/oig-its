0 Prerequisites & Setup

Before starting, ensure the system is running.

Start Docker Services

Open PowerShell in the project directory and start all backend services:

docker compose up -d
5 services should be running: postgres, redis, minio, meilisearch, mailhog

Start the Application

npm run dev
Server starts at http://localhost:3000

Seed the Database (first time only)

npx prisma migrate dev && npx prisma db seed
200 cases, 150 subjects, 800 documents, 10,000 audit logs seeded

3 Case Management

Complete lifecycle from intake to closure with 9 case types and 7 statuses.

Browse Cases

http://localhost:3000/dashboard/cases

DataTable with columns: Case #, Title, Status, Priority, Type, Assigned To, Due Date. Use filters for Status, Type, Priority. Search by case number or title.

SA Johnson sees 12 assigned active cases (RBAC filtering)

Create a New Case

http://localhost:3000/dashboard/cases/new

3-step wizard:

  1. Details: Title (min 5 chars) + Description
  2. Classification: Case Type (9 options) + Priority + Due Date
  3. Review: Confirm and submit
Auto-generates case number OIG-2026-XXXXX, auto-assigns you as lead investigator

View Case Detail

Click any case to see the 10-tab detail view:

Overview

Description, counts, details sidebar, team

Documents

Upload, download, approve, version

Evidence

Chain of custody, exhibit numbers

Tasks

Board & list view, assignment

Subjects

People & orgs with roles

Timeline

Auto-generated event chronology

Violations

Charges per subject

Financial

Recoveries, fines, savings

Techniques

Interviews, warrants, surveillance

Referrals

DOJ, FBI, state agencies

Case Status Progression

Progress a case through its lifecycle:

INTAKE → OPEN → ACTIVE → UNDER_REVIEW → PENDING_ACTION → CLOSED → ARCHIVED

Pre-close validation requires: at least 1 subject, 1 document, all tasks completed, and checklist items done

Case Locking

Login as Admin or Supervisor to lock a case. Locked cases return HTTP 423 on any edit attempt. Cases auto-lock when closed.

Case Relationships

Link related cases (spinoffs, merged investigations) from the case detail API. Related case numbers and titles are preserved.

Draft Cases

Save incomplete cases as drafts with relaxed validation. Auto-titled "Draft - OIG-2026-XXXXX".

5 Evidence & Chain of Custody

Evidence Types

8 types: Document, Photo, Video, Audio, Digital, Physical, Testimony, Other. Each gets auto-labeled with exhibit numbers (EX-001, EX-002).

Chain of Custody

Every evidence transfer is tracked with:

  • From User → To User
  • Action (Collected, Transferred, Reviewed, Stored, Released)
  • Timestamp and notes
Immutable chain — entries cannot be modified or deleted

6 Document Management

Upload Documents

Navigate to any case → Documents tab. Upload files up to 50 MB in 30 supported formats (PDF, Word, Excel, PowerPoint, images, audio, video, email).

Files stored in MinIO (S3-compatible). Access logged for every download.

Document Versioning

Upload a new version of an existing document. Version numbers auto-increment (v1, v2, v3...). Previous versions are preserved.

Supervisory Approval

Flag documents as requiring approval. Supervisors/Admins get notified and can approve or reject. Document status: DRAFT → UPLOADED → REVIEWED → APPROVED.

Document Templates

19 predefined templates auto-populate with case data:

Subpoena, Memorandum of Interview, Report of Investigation, Evidence Receipt, Witness Statement, Case Closure Memo, Affidavit, Search Warrant, Arrest Warrant, Grand Jury Subpoena, Deposition Notice, Case Transfer Memo, Investigative Summary, Referral Letter, Suspension Notice, Complaint Form, Evidence Log Sheet, Interview Schedule, Travel Authorization

ZIP Download

Download all case documents as a single ZIP archive from the API.
