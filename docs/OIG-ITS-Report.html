🤖 AI Algorithms (22)

18 statistical/rule-based algorithms implemented in pure TypeScript plus 4 Claude LLM-powered features. All accessible via the AI Insights dashboard.

Statistical & Rule-Based (18)

Statistical — Z-Score

1. Anomaly Detection

Flags financial outliers (amount > mean + 2σ), unusually long cases, and abnormal user activity patterns. Three sub-detectors: financial, case, activity.

GET /api/ai/anomalies

Rule Engine — Keyword Matching

2. Risk Scoring

Scores complaints 0-100. Factors: fraud keywords (+20), source weight (whistleblower +15), contact info (+10), description quality (+5-15). Auto-runs on hotline intake.

POST /api/ai/risk-score

Statistical — Linear Regression

3. Predictive Analytics

Predicts case duration, closure outcome (substantiated vs not), monthly caseload forecast, and identifies at-risk cases likely to miss deadlines.

GET /api/ai/predictions

ML — Cosine Similarity

4. Case Similarity

22-dimensional feature vectors (type, priority, source one-hot + numeric counts). Cosine similarity finds top N similar cases with score 0.0-1.0.

GET /api/ai/similar-cases?caseId=

ML — K-Means Clustering

5. Case Clustering

Groups all active cases into k=5 clusters using Euclidean distance on feature vectors. Labels clusters by most common type and source.

GET /api/ai/clusters

NLP — Levenshtein + Soundex

6. Duplicate Entity Resolution

Fuzzy matching: Levenshtein edit distance (<3 = match), Soundex phonetic codes, email domain, address similarity. Confidence 0-1 per pair.

GET /api/ai/duplicate-subjects

Graph — BFS Traversal

7. Network Analysis

Builds subject-case graph. BFS finds connected components. Degree analysis identifies hub entities. Detects fraud rings (subjects sharing 2+ cases).

GET /api/ai/network

Rule Engine — Regex + MIME

8. Document Classification

Auto-categorizes into 8 types (subpoena, interview memo, financial record, etc.) via keyword regex. Extracts tags (amounts, dates, agencies). Runs on every upload.

POST /api/ai/classify-document

Multi-Factor Scoring

9. Investigator Recommender

Scores investigators: expertise match (30pts), workload (25pts), success rate (25pts), availability (20pts). Returns top 3 with factor breakdowns.

POST /api/ai/recommend-investigator

Rule Engine — Threshold

10. Auto-Escalation

Scans active cases: financial >$500K → CRITICAL, due <7 days → HIGH, no activity 14 days → stale, evidence without violations → review needed.

GET /api/ai/escalations

Weighted Scoring — A-F Grade

11. Evidence Strength

0-100 score: per-type points (physical +5, digital +4), custody completeness (+10), type diversity (+15), source corroboration (+10), timeline coverage (+5).

GET /api/ai/evidence-strength?caseId=

Gap Analysis — Timeline

12. Timeline Anomalies

Detects: activity gaps >30 days, post-review evidence collection, weekend uploads, rapid same-day status changes. Severity-rated per anomaly.

GET /api/ai/timeline-anomalies?caseId=

Checklist Scoring — 13 Criteria

13. Closure Readiness

100-point checklist: subjects (15), violations (10), evidence (10), financials (10), tasks (10), checklist (10), referrals (5), techniques (5), notes (5), etc. Ready at ≥80.

GET /api/ai/closure-readiness?caseId=

NLP — Jaccard + Levenshtein

14. Complaint Deduplication

60% subject similarity (Levenshtein) + 40% keyword overlap (Jaccard coefficient). Cross-matches against inquiries and open cases. Auto-runs on hotline intake.

POST /api/ai/complaint-dedup

Statistical — Std Deviation

15. Workload Balancing

Per-investigator: active cases (×3) + critical (×5) + overdue tasks (×4). Flags overloaded (>avg+1.5σ) and underloaded (<avg-1σ). Supervisor queue depth.

GET /api/ai/workload

Pattern Mining — IQR

16. Financial Patterns

Detects: round numbers ($10K/$50K/$100K), just-below-threshold ($9,999), sequential amounts per subject, IQR outliers, weekend transactions.

GET /api/ai/financial-patterns

Multi-Factor — 0-100 Risk

17. Subject Risk Profiling

Cases (+5 each), violations (+10/+20 substantiated), financial impact (log scale), network hub score, repeat offender (+15), role escalation (+10).

GET /api/ai/subject-risk

Template Generation

18. Case Narrative

Generates plain-English investigation summary from structured data: opening, subjects, techniques, violations, financial impact, referrals, status.

GET /api/ai/case-narrative?caseId=

POST /api/ai/analyze-document

Claude Sonnet 4 — LLM

21. Interview Question Generator

Generates targeted questions by case type and subject role: opening, substantive, probe, and closing categories with interview tips.

POST /api/ai/interview-questions

Claude Sonnet 4 — LLM

22. Report Generation

Generates professional investigation reports (summary, narrative, findings, recommendation) from complete case data using AI writing.

POST /api/ai/generate-report

Security Features

  • 8-hour JWT session timeout
  • Account lockout after 5 failed login attempts (15-minute cooling)
  • bcryptjs password hashing with salted hashes
  • IP address whitelisting via middleware (configurable CIDR ranges)
  • Password reset with time-limited email tokens
  • Field-level edit permissions by role × case status matrix
  • Complete audit trail with field-level change tracking (old → new values)
  • Document access logging (every download tracked)
  • Immutable chain of custody for evidence
  • Case auto-lock on closure
  • Records retention with soft-delete (configurable years)
